# Threat Hunt Report: H013 — PowerShell (T1059.001) — Execution

## Correction notice

This version replaces the original generated report. The review found four material problems:

1. The folder query contained generic terms (`often`, `utilize`, `powerful`, `language`, `available`, `windows`) and retrieved telemetry unrelated to PowerShell.
2. Several unrelated Sigma matches were written as if they supported the PowerShell hypothesis.
3. Model citations such as `51-55 (EventID-1116 and 1117 entries)` and comma-separated references were not accepted by the original verifier, so the claimed findings were not verified.
4. The raw evidence sample was dominated by large Event ID 4799 XML records unrelated to the hunt and was not useful for review.

The original case and approval remain valid audit artifacts because the original findings failed verification. This correction does not silently approve or close them.

This query is not sufficiently specific for a PowerShell conclusion. Future runs now remove generic prose terms and retain explicit indicators such as `powershell`, `powershell.exe`, and `4104`.

## Corrected findings

### 1. PowerShell telemetry was present, but malicious use was not established

- The deterministic THOS rule `thos-0001` (PowerShell Script Block Logging Execution) matched **2 records**.
- A 4104 match confirms script-block telemetry exists; it does not by itself prove the script was malicious.
- The original report did not retain a compact, verifier-approved citation to the exact script content, host, user, and parent process. Those records must be reviewed before assigning malicious intent.
- **Assessment:** partially supports visibility for the hypothesis; insufficient evidence for compromise.

### 2. Other detections are contextual and outside the primary hypothesis

The retrieved dataset also produced Windows Defender, audit-log clearing, token/UAC, Rundll32, Regsvr32, WinRM, MSI, LSASS-access, DLL-hijacking, reverse-shell, and JScript rule matches. These may warrant separate triage, but the broad query does not establish temporal, host, user, or process-lineage correlation with the two PowerShell 4104 records.

They are therefore **not accepted as evidence of malicious PowerShell activity in this report**.

### 3. Original “hard-evidence” statements were not verified

The original report claimed that Defender blocked multiple threats, the Security log was cleared, token duplication/UAC bypass occurred, and Rundll32 was used maliciously. Its verifier failed with invalid reference expressions. Those statements must be treated as unverified leads, not confirmed findings, until the cited records are examined directly.
